# Bazaar Arena 图形界面说明

本文档描述当前阶段软件 GUI 的结构与行为，便于维护与后续扩展。

---

## 总体结构

- **主窗口**：`MainWindow`，启动时显示，负责卡组集管理与卡组编辑。
- **子窗口/对话框**：单次模拟、批量模拟、卡组另存为，均以模态或独立窗口方式从主窗口打开。
- **技术**：WPF (XAML + C#)，界面与注释使用中文。

---

## 主窗口 (MainWindow)

### 菜单栏

- **文件(_F)**
  - **新建卡组集**：弹出“另存为”对话框（初始目录为 `Data/Decks`），用户选择路径后创建空卡组集 JSON 并切换为该文件；取消则不操作。
  - **打开卡组集...**：打开文件对话框（初始目录为 `Data/Decks`），选择 JSON 后加载卡组集，左侧列表切换为该文件内的所有卡组。
  - **退出**：关闭应用。

- **模拟(_S)**
  - **单次模拟...**：打开单次模拟窗口。
  - **批量模拟...**：打开批量模拟窗口。

### 左侧：卡组列表

- 标题「卡组列表」，下列表框显示当前卡组集中所有卡组（显示为「[等级] 卡组名」）；顺序与卡组集 JSON 中 `decks` 数组一致。
- **新建**：清空编辑区，进入“新建卡组”状态；保存时需通过“另存为”输入新卡组名并写入当前卡组集。
- **重命名**：对当前选中卡组重命名（弹窗预填当前名），成功后自动保存卡组集。
- **上移 / 下移**：调整选中卡组在列表中的位置（第二行按钮），操作后自动保存卡组集。
- **删除**：删除当前选中卡组（需确认），并自动保存卡组集到当前关联文件（若有）。

点击列表项会在右侧编辑区打开该卡组。

### 右侧：编辑区或欢迎语

- 未选卡组时显示欢迎语。
- 选中或新建卡组后显示编辑区：
  - 玩家等级 (1–20)
  - 槽位摘要（已用/上限）
  - 卡组槽位网格（可拖拽排序；**拖出卡组区域**即移除，无需单独“删除”区）。第三行为物品名称，居中显示（含跨列居中）。
  - 物品池（拖拽到卡组添加）
  - **悬停说明**：鼠标在卡组内物品或物品池上悬停约 400ms 后显示 Tooltip，三行：名称（卡组内按等级着色）、冷却时间、物品描述（Desc，支持占位符如 `{Damage}`，关键词与日志同色）。
  - **保存**：将当前编辑保存到当前卡组（若未有关联卡组则等同于另存为），并自动保存卡组集到文件（若有路径；若无则弹出另存为选路径）。
  - **另存为...**：弹出“另存为”对话框（居中于主窗口、输入框自动聚焦），输入新卡组名后保存到当前卡组集并自动写回卡组集文件。

### 标题栏

- 无关联文件时：「Bazaar Arena - 未命名」
- 有关联文件时：「Bazaar Arena - 文件名」

### 启动行为

- 若存在 `Data/Decks/default.json`，启动时自动加载为该卡组集并刷新列表；加载失败则保持空列表。

## 单次模拟窗口 (SingleSimulateWindow)

- **标题**：单次模拟；**Owner**：主窗口；居中显示。
- **内容**：
  - 玩家1/玩家2 卡组下拉框（来自当前卡组集）。
  - 「开始模拟」按钮。
  - 结果摘要（胜方与用时）。
  - 左侧：战斗过程日志（RichTextBox，关键词着色）。
  - 右侧：每物品统计（DataGrid：玩家、物品、使用、伤害、灼烧、剧毒、护盾、治疗、再生）。
  - 下方：生命曲线（Canvas 绘制，玩家1 蓝色、玩家2 红色；阶梯状、仅生命变化时打点；负生命显示为 0）。

---

## 批量模拟窗口 (BatchSimulateWindow)

- **标题**：批量模拟；**Owner**：主窗口；居中显示。
- **内容**：
  - 玩家1 卡组、玩家2 卡组（各占一行，下拉框）。
  - 模拟场次（默认 100，1–100000）。
  - 「开始批量模拟」按钮。
  - 进度条与进度文字。
  - 结果：玩家1 胜率、玩家2 胜率、平局率，分三行显示。

---

## 卡组另存为对话框 (SaveAsDialog)

- **标题**：另存为；**Owner**：主窗口；**WindowStartupLocation**：CenterOwner。
- 输入卡组名称的文本框，打开时自动聚焦并全选。
- 确定 / 取消；确定后主窗口用该名称将当前编辑保存到当前卡组集。

---

## 卡组集与文件

- **格式**：一个 JSON 文件 = 一个卡组集；内含 `version` 与 `decks` 数组，每项为 `id` + `deck`（卡组数据）。
- **默认路径**：打开/保存卡组集时文件对话框的初始目录为 `Data/Decks`（由 `App.DecksDirectory` 提供）。
- **自动保存**：在以下操作后会尝试将当前卡组集写回关联文件（若无路径则弹出另存为）：
  - 保存卡组
  - 另存为卡组
  - 删除卡组
  - 重命名卡组、上移/下移卡组

---

## 窗口与对话框约定

- 子窗口使用 `Owner = this`（主窗口）以便居中（CenterOwner）与层级关系正确。
- 卡组另存为等小型对话框在 Loaded 时用 `Dispatcher.BeginInvoke(Loaded)` 对输入框 `Focus()` 与 `SelectAll()`，保证打开即可输入。
- Tooltip 为减轻卡顿：卡组内物品在 `ToolTip.Opened` 时再构建内容；物品池在 `ToolTipService.ToolTipOpening` 时构建，并按物品名缓存。`InitialShowDelay` 约 400ms。
