# Bazaar Arena 重构方案

## 重构目标

1. 最大化计算性能
2. 简化物品定义流程，避免 AI 协同开发中无法正确理解用户设计的问题
3. 优化 GUI 显示，提高用户交互体验
4. 对现有框架中的遗留错误做出修正

## 重构计划

1. 使用 YAML 重写物品定义文件
2. 使用 C++ 重写核心部分（数据库 + 对战模拟器）
3. 使用 Vue.js 重写前端 GUI 界面，使用 Web 实现 GUI
4. 使用 SQLite 保存数据库用于 Web 应用程序
5. 使用 Flask 实现 Web 后端，调用 C++ 的 exe（结果文件写入 JSON），通过 Restful API 向前端发送计算结果 JSON

## 分层原则

整个重构后的程序分为若干层，每一层的功能有严格的分界。

1. 前端层（Vue）：负责显示和用户操作，不进行任何计算，用户请求发回给后端（主要由 AI 完成）
2. 后端层（Flask）：负责响应前端的请求并维护数据库，只负责显示级的计算，在数据库里的内容（物品信息、卡组信息等）会读取数据库并返回给前端，需要计算的内容（如对战模拟）会调用 C++ 的 exe，等待完成后读取结果文件的 JSON 返回给前端（主要由 AI 完成）
3. 计算层（C++）：负责实际进行对战模拟（人工定框架，和 AI 协同完成）
   1. 调试层（C++）：用于为 C++ 程序调试使用提供 CLI（主要由 AI 完成），在 CLI 中代替前端、后端层
4. 数据层（YAML）：负责定义物品（人工完成）
   1. 格式转换层（Python）：将 YAML 数据通过脚本转换为 C++ 代码以供计算层使用，以及将其显示部分写入 SQLite 数据库以供后端层使用
